git is an open source version control system. what does that mean? open source to me can mean many things. open source for git can mean that anyone who has interest and time can contribute to the code that makes this system work and all of the code is available for you to download and make into your own. the code societies blog is open source too. open source to me is kind of a philosophical question in that what does it truly mean for code to open? how transparent does it need to be in order for it to be considered open? im thinking about facebook claiming the word open source but also only sharing very select parts of their code and capitalizing on the free labor of people who have time to spend and knowledge of coding in the first. there's also the question of what else can be open source? can code societies as a workshop be open source? what structures and agreements need to be in place in order for something to be open source? also thinking about things like open source journalism or open source fonts. how open source might just be a kind of generous knowledge sharing and collaborative work.

version control means that when you use it with any kind of file it will keep track of any and all changes you make to those files. so if you are working on a poem inside of a txt file git will be able to keep track of every single change you make, an extra space, a new word etc. the amazing thing is that you will also be able to go back and revert to previous versions of the thing you are making.

it also makes it so that multiple people can work on the same file without worrying too much about overwriting each others work, or at least overwriting it in a way that is recorded.

github is basically a graphical user interface hosted on the web which makes it easy for people to access and contribute to each others code

forking is a very common method for contributing to open source code. it is basically creating a copy of someone elses code on their account and moving it to your account. so you can now use their code as a base and make any edits your heart desires because you are now working on code that is under your account. this is cool if you want to use someone code as a starting point for your own project or if you want to contribute your own changes to someones project which is what we will be doing.
